# Glosario — GestPro CRM

> Términos técnicos, inmobiliarios y de negocio utilizados a lo largo del proyecto.

---

| Término | Definición |
|:--------|:-----------|
| **Agente Inmobiliario** | Profesional especializado en la intermediación de transacciones de compra, venta y alquiler de bienes raíces, actuando como enlace entre compradores y vendedores. |
| **API** *(Application Programming Interface)* | Conjunto de protocolos y herramientas que permite la comunicación entre diferentes componentes de software o sistemas externos. |
| **Arrendamiento** | Contrato mediante el cual el propietario cede el uso de una propiedad a un inquilino a cambio de una renta periódica. |
| **Avalúo** | Proceso técnico de valoración profesional de una propiedad para determinar su valor comercial en el mercado actual. |
| **Backup** | Copia de seguridad automática de los datos del sistema que se realiza periódicamente para prevenir pérdida de información. |
| **Base de Datos** | Sistema organizado de almacenamiento digital que mantiene toda la información de propiedades, clientes, transacciones y usuarios del CRM. |
| **BI** *(Business Intelligence)* | Conjunto de métricas, reportes y visualizaciones que permiten analizar el rendimiento del negocio inmobiliario desde el dashboard. |
| **Bien Raíz** | Propiedad inmueble que incluye terrenos, edificios y cualquier estructura permanente anexa al suelo. |
| **Brochure Digital** | Documento PDF generado automáticamente con las características, fotografías y precio de una propiedad, listo para compartir con clientes. |
| **Cliente Potencial** *(Lead)* | Persona o entidad que ha mostrado interés inicial en servicios inmobiliarios pero aún no ha realizado una transacción. |
| **Comisión** | Porcentaje del valor de una transacción inmobiliaria que recibe el agente como compensación por sus servicios profesionales. |
| **CRM** *(Customer Relationship Management)* | Sistema de gestión de relaciones con clientes que centraliza información, interacciones y procesos comerciales. |
| **CRUD** | Operaciones básicas de sistemas de bases de datos: **C**reate (Crear), **R**ead (Leer), **U**pdate (Actualizar), **D**elete (Eliminar). |
| **Dashboard** | Panel de control visual que presenta métricas, indicadores clave y resúmenes ejecutivos del negocio inmobiliario en tiempo real. |
| **Downline** | Estructura jerárquica de subordinados de un agente; define la visibilidad de datos por rol (JUNIOR: solo propios; SENIOR: su downline completo). |
| **DPI** *(Documento Personal de Identificación)* | Documento oficial de identificación único utilizado en Guatemala para verificar la identidad de personas. |
| **Escritura Pública** | Documento legal que formaliza oficialmente la transferencia de propiedad de un bien inmueble. |
| **Estatus de Propiedad** | Estado actual de una propiedad en el inventario. Valores posibles: `Disponible`, `Reservada`, `Vendida`, `Rentada`, `Retirada del mercado`. |
| **FONAPAZ** | Fondo Nacional para la Paz de Guatemala; entidad que maneja ciertos aspectos registrales de propiedades. |
| **Frontend** | Interfaz de usuario visible y funcional con la que interactúan directamente los agentes inmobiliarios y administradores. |
| **Geocodificación** | Conversión de una dirección textual a coordenadas geográficas (latitud/longitud) para ubicar propiedades en el mapa. |
| **GTQ** *(Quetzal Guatemalteco)* | Moneda oficial de Guatemala utilizada para todas las transacciones y valoraciones en el sistema. |
| **Historial de Cliente** | Registro cronológico completo de todas las interacciones, visitas, preferencias y transacciones asociadas a un cliente específico. |
| **Hosting** | Servicio de alojamiento web en servidores en la nube que mantiene el sistema CRM accesible 24/7. |
| **ICS** *(iCalendar)* | Formato estándar de archivo de calendario (.ics) generado al confirmar una visita, compatible con Google Calendar, Outlook y Apple Calendar. |
| **Inmueble** | Bien inmobiliario que no puede moverse de su ubicación (casas, apartamentos, terrenos, oficinas, locales comerciales). |
| **Inquilino** | Persona que ocupa una propiedad mediante contrato de arrendamiento pagando una renta mensual. |
| **ISO/IEC 25010** | Estándar internacional que define un modelo de calidad para sistemas y software, especificando características como usabilidad, confiabilidad y rendimiento. |
| **JSON** *(JavaScript Object Notation)* | Formato ligero de intercambio de datos utilizado para la comunicación entre frontend y backend. |
| **JWT** *(JSON Web Token)* | Estándar para transmitir credenciales de autenticación entre cliente y servidor de forma compacta y verificable mediante firma digital. |
| **KPI** *(Key Performance Indicator)* | Indicadores clave de rendimiento que miden la efectividad y éxito del negocio inmobiliario. |
| **Lead** | Cliente potencial que ha mostrado interés inicial pero requiere seguimiento comercial para convertirse en cliente activo. Ver también: *Cliente Potencial*. |
| **Listing** | Registro público de una propiedad disponible para venta o alquiler con todas sus características y precio. |
| **MLS** *(Multiple Listing Service)* | Sistema compartido donde múltiples agentes pueden listar y buscar propiedades disponibles. |
| **MTBF** *(Mean Time Between Failures)* | Tiempo promedio entre fallas del sistema; métrica de confiabilidad técnica. |
| **MTTR** *(Mean Time To Repair)* | Tiempo promedio para reparar y restaurar el servicio después de una falla. |
| **Multi-tenant** | Modelo arquitectónico donde una sola instancia del sistema sirve a múltiples *tenants* simultáneamente, con datos completamente separados entre ellos. |
| **Notificación Automática** | Alerta generada automáticamente por el sistema para informar sobre eventos importantes (nueva visita, cambio de precio, vencimiento de documento, etc.). |
| **Oferta** | Propuesta formal de compra o alquiler presentada por un cliente para una propiedad específica. |
| **Onboarding** | Proceso inicial de configuración de una nueva agencia (*tenant*) en el sistema: creación de cuenta, configuración de roles y carga de datos base. |
| **Pipeline de Ventas** | Proceso estructurado que sigue una oportunidad comercial desde el contacto inicial hasta la transacción cerrada. |
| **Propietario** | Persona física o jurídica que posee legalmente un bien inmueble registrado a su nombre. |
| **PYMES** | Pequeñas y Medianas Empresas; segmento objetivo principal del sistema. |
| **Query** | Consulta estructurada a la base de datos para recuperar información específica de propiedades, clientes o transacciones. |
| **RBAC** *(Role-Based Access Control)* | Sistema de control de acceso basado en roles que determina qué funciones puede realizar cada usuario. |
| **Registro de la Propiedad** | Institución oficial guatemalteca que certifica y registra la propiedad legal de bienes inmuebles. |
| **RLS** *(Row-Level Security)* | Mecanismo de PostgreSQL que restringe qué filas puede ver o modificar cada sesión; es el núcleo del aislamiento de datos entre tenants. |
| **ROI** *(Return on Investment)* | Retorno de inversión; métrica que evalúa la rentabilidad de las operaciones inmobiliarias. |
| **SaaS** *(Software as a Service)* | Modelo de entrega de software en la nube donde los clientes pagan una suscripción en lugar de instalar el sistema localmente. |
| **SAT** *(Superintendencia de Administración Tributaria)* | Entidad gubernamental guatemalteca responsable de la recaudación de impuestos sobre propiedades. |
| **Sindicación** | Publicación automática o manual de una propiedad en portales externos (sitio web público, redes sociales) a partir de su ficha interna en el CRM. |
| **Stakeholder** | Cualquier persona o entidad que tiene interés o se ve afectada por el proyecto del sistema CRM. |
| **Tenant** | Cada agencia inmobiliaria que contrata el servicio; opera de forma completamente aislada de los demás tenants dentro de la misma instancia del sistema. |
| **Throughput** | Cantidad de transacciones o usuarios que el sistema puede procesar simultáneamente sin degradación de rendimiento. |
| **TOTP** *(Time-based One-Time Password)* | Algoritmo de segundo factor de autenticación (2FA) que genera un código temporal cada 30 segundos. |
| **Trazabilidad** | Capacidad del sistema de rastrear y registrar todas las acciones y cambios realizados para fines de auditoría. |
| **UML** *(Unified Modeling Language)* | Lenguaje de modelado unificado utilizado para documentar la arquitectura y procesos del sistema. |
| **Uptime** | Porcentaje de tiempo que el sistema permanece operativo y accesible para los usuarios. |
| **Visita** | Cita programada entre un agente, cliente y propiedad para la inspección física del inmueble. |
